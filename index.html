并发编程之辅助工具类

发表于 2019-03-24

前言：
在一次迭代开发中，有一个功能是需要去计算各个部分的汇总值，然后再进行合计，最终通知到用户。然后由于数据量大，需要使用多线程去处理，
问题是怎么控制所有子部分计算完毕后，再汇总通知到用户。其实这里是涉及到最后一步如何同步处理，我选择使用了CountDownLatch。在java 1.5中，提供了一些非常有用的辅助类来帮助我们进行并发编程，比如CountDownLatch，CyclicBarrier和Semaphore，今天我们就来学习一下这三个同步辅助类的用法。

Synchronization Aid：同步辅助器

下面表格，简单阐述下各自的特点

类名称 解释
CountDownLatch 是一个同步的辅助类，允许一个或多个线程，等待其他一组线程完成操作，再继续执行
CyclicBarrier 是一个同步的辅助类，允许一组线程相互之间等待，达到一个共同点，再继续执行。
Semaphore 是一个同步的辅助类，用来控制同时访问特定资源的线程数量，它通过协调各个线程，以保证合理的使用公共资源。

三个辅助类，都位于java.util.concurrent包，可以帮助解决不少并发上的棘手问题，下面的demo可以看到各自的使用场景。

软件的可维护性和可复用性

知名软件大师Robert C.Martin认为一个可维护性(Maintainability) 较低的软件设计，通常由于如下4个原因造成：

  • 过于僵硬(Rigidity)

  • 黏度过高(Viscosity)

  • 灵活性(Flexibility)

大话设计模式之单例模式

前言：
在看《spring源码深度解析》这本书时，讲到spring中bean的生命周期，提到了bean默认是单例模式的。可通过修改配置，可以变为多实例模式。
然后又涉及到单例模式在并发下是否安全的问题。为了解决内心的疑惑，就整合一些资料和自己的理解，来讲下常用的设计模式-单例模式。

定义

作为对象的创建模式，单例模式确保某一个类只有一个实例，而且自行实例化并向整个系统提供这个实例。这个类称为单例类。

结构特点

  • 单例类只能有一个实例。
  • 单例类必须自己创建自己的唯一实例。
  • 单例类必须给所有其他对象提供这一实例。

kafka 简述

发表于 2019-03-11 | 更新于 2019-03-24

Kafka是什么？

     Kafka 是一个高吞吐量、分布式的发布—订阅消息系统。

  • 它最初由LinkedIn公司开发，后来成为Apache 项目的一部分。Kafka 核心模块使用Scala 语言开发，支持多语言（如Java、C/C++、Python、Go、Erlang、Node.js 等）客户端，它以可水平扩展和具有高吞吐量等特性而被广泛使用。
  • 当前的Kafka 已经定位为一个分布式流式处理平台（a distributed streaming platform），目前越来越多的开源分布式处理系统（如Flume、Apache Storm、Spark、Flink等）支持与Kafka 集成。

kafka背景

     随着信息技术的快速发展及互联网用户规模的急剧增长，计算机所存储的信息量正呈爆炸式增长，目前数据量已进入大规模和超大规模的海量数据时代，如何高效地存储、分析、处理和挖掘海量数据已成为技术研究领域的热点和难点问题。

Spring Boot简述

发表于 2019-03-03 | 更新于 2019-03-24

Spring Boot是什么？

Spring Boot是解决这个问题的方法。Spring Boot已经建立在现有spring框架之上。使用spring启动，我们避免了之前我们必须做的所有样板代码和配置。
因此，Spring Boot可以帮助我们以最少的工作量，更加健壮地使用现有的Spring功能。

优点

  • 减少开发，测试时间。
  • 使用JavaConfig有助于避免使用XML。
  • 避免大量的Maven导入和各种版本冲突。
  • 通过提供默认值快速开始开发。
  • 没有单独的Web服务器需要，这意味着你不再需要启动Tomcat或其他任何容器。
  • 需要更少的配置 因为没有web.xml文件。只需添加用@ Configuration注释的类，然后添加用@Bean注释的方法，Spring将自动加载对象并像以前一样对其进行管理。您甚至可以将@Autowired添加到bean方法中，以使Spring自动装入需要的依赖关系中。
  • 基于环境的配置 使用这些属性，您可以将您正在使用的环境传递到应用程序：-Dspring.profiles.active = {enviornment}。在加载主应用程序属性文件后，Spring将在（application{environment} .properties）中加载后续的应用程序属性文件。
  • 有了Spring Boot可以非常方便集成现在流行的Spring Cloud微服务框架。
